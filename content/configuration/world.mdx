# World, Blips & Props

Client-side world configuration.

## Map blips — `configs/cfg-blips-cl.lua`

Static map blips for businesses, government, transport and public spaces.

```lua
blips = {
    { category = "Business", type = "Bank", id = 108, x = -109.7, y = 6463.6,
      name = "Blaine County Savings Paleto", image = "bcs_paleto" },
    { category = "Business", type = "Gun Store", id = 110, x = 1697.7, y = 3753.2,
      name = "Ammunation Sandy Shores", image = "ammunation_sandy",
      phone = "1-999-WEAPONS", bleeter = "@Ammunation" },
}
```

| Field | Meaning |
| --- | --- |
| `category` | `Business`, `Government`, `Public Transport`, or `Public Space`. |
| `type` | Sub-type label. |
| `id` | GTA blip sprite id. |
| `x` / `y` | World position. |
| `name` | Blip name. |
| `image` | Streamed `.ytd` texture shown top-left on hover (optional). |
| `phone` / `bleeter` | Optional contact tags. |
| `location` | Optional map region (e.g. Cayo Perico); defaults to San Andreas. |

## Areas — `configs/cfg-areas-cl.lua`

Named world areas/regions used for location display (street/area names). `AreaDefinitions` groups
`AreaPoints` (coordinate vertices) into named areas and regions:

```lua
AreaDefinitions = {
    { area = "Sandy Shores", region = "San Tierra", configpoints = { 1, 2, 3, 4 } },
}

AreaPoints = {
    { x = 2296.43, y = 3773.26 },
    { x = 1359.96, y = 3254.03 },
    -- …
}
```

## Weapon-on-back — `configs/cfg-weapons-cl.lua`

Positions weapons on players' backs (if enabled client-side). `weaponOnBackSettings` sets the bone,
offset and rotation, plus per-weapon prop hashes:

```lua
weaponOnBackSettings = {
    back_bone = 24816,
    x = 0.075, y = -0.15, z = -0.02,
    x_rotation = 0.0, y_rotation = 165.0, z_rotation = 0.0,
    compatable_weapon_hashes = {
        ["w_me_bat"]            = -1786099057,
        ["prop_ld_jerrycan_01"] = 883325847,
        -- …
    },
}
```

## Spawn selector — `configs/cfg-spawns-sh.lua`

A map-based spawn picker: the player sees the city map with pins and chooses where to start, instead of
one fixed spawn. Gated by `Features.spawnselect` (off = the classic single `fallback` spawn).

```lua
SpawnCfg = {
    enabled     = true,
    onFirstJoin = true,   -- show the picker after connecting
    onRespawn   = true,   -- show it again after death, once released
    timeout     = 60,     -- seconds before the fallback is used anyway (0 = wait forever)
    fallback    = { x = 2728.5073, y = 1373.4258, z = 24.5227, heading = 0.0 },  -- skipped / timed-out / error

    points = {
        { id = "hospital", label = "Pillbox Hill Medical", info = "Hospital entrance",
          x = 298.50, y = -584.70, z = 43.26, heading = 70.0 },
        { id = "coroner", label = "Coroner's Office", info = "County services",
          x = 240.40, y = -1379.90, z = 33.80, heading = 320.0, jobs = { "Coroner" } },  -- job-gated
        { id = "paleto_jail", label = "Bolingbroke Release", info = "Blaine County",
          x = 1855.81, y = 2601.95, z = 45.32, heading = 270.0, aopOnly = true },  -- only inside the AOP
    },
}
```

Each point takes `id` (the client sends **this**, never coordinates — the server re-checks the job + AOP
before it hands back a position), `label`, optional `info`, an optional `jobs` list (omit = everyone), and
`aopOnly` (only offered while the point is inside the current Area of Play). Capture your own points in-game:
stand where players should appear, face the right way, then run `/spawnpoint <id> <label>` — it prints a
ready-to-paste line. It is always **fail-safe**: anything unexpected falls back to `fallback`, so no one
gets stuck on a black screen.

<Callout type="info">
  Map blips, areas and weapon-on-back are cosmetic/world lists — none are required to run LACORE. Leave
  them empty to disable the feature, or populate them for your map.
</Callout>
