# Anticheat & Security

## Anticheat — `configs/cfg-anticheat-sh.lua`

<Callout type="error">
  **Do not enable without configuring it first.** `Anticheat.Enabled = false` by default. Read every
  threshold below and tune it to your server before switching it on.
</Callout>

Per-detection action:

| Action | Effect |
| --- | --- |
| `"log"` | Discord webhook (`Webhooks.AdminLog`) only. |
| `"kick"` | Webhook + kick. |
| `"ban"` | Webhook + permanent ban (via `BanPlayer`). |

Bypass: `lacore_devmode = true` disables it entirely; players with the `Staff` / `Dev` Discord role, or
the `lacore.ac.bypass` ACE permission (`Anticheat.Whitelist`), are never punished (only logged).

```lua
Anticheat = {
    Enabled = false,

    Explosions = {
        action = "ban",
        blocked = {          -- explosion types a client can never legitimately trigger
            [30] = true,     -- buried mine
            [59] = true,     -- orbital cannon
            [72] = true,     -- script missile
            -- …
        },
        ratePer10s = 8,      -- more than X explosions in 10s → punish
    },

    WeaponDamage = {
        action    = "kick",
        maxDamage = 250,     -- above this = damage modifier (sniper headshot ≈ 200)
    },

    EntitySpam = {
        action       = "kick",
        maxPerMinute = 20,   -- max client-spawned entities per minute
    },
    -- … excerpt only — the file also carries GodMode, SpeedHack, Teleport, NoClip, Invisible, SuperJump,
    -- BlacklistedWeapons, BlacklistedVehicles (delete = true), AntiDump, Honeypots, EventRateLimit, a Trust
    -- escalation model (kickAt 8 / banAt 20), the server-authoritative ServerSweep + Combat checks, and a
    -- recommended FXServer convar block (sv_scriptHookAllowed 0, sv_pureLevel 2, sv_enforceGameBuild, …).
}
```

## Member-only vehicles — `configs/cfg-blacklist-sh.lua`

Vehicle **classes** and specific **models** only "members" may drive:

```lua
memberOnlyVehicles = {
    18, 16, 15, 19, 7, 22,   -- vehicle CLASSES (see FiveM vehicle class list)
    -- specific models:
    "kosatka", "technical", "terbyte",   -- armoured / armed
    "bestiagts", "corsita",              -- rare / expensive
    "monster", "rcbandito",              -- abuse-prone niche
    -- …
}
```

Add class **numbers** or model **name strings**. Who counts as a "member" is set by
`AccessControl.membership` in [`cfg-server-sv.lua`](/configuration/discord) — `"auto"` / `"open"` / `"role"`,
driven by the Discord **Member** role, not by playtime. (`ProfileConfig.membershipHours` only decides when
the "you've played enough to apply for membership" notice fires — it does **not** unlock these vehicles.)

## Dashboard remote-config gate — `configs/cfg-remoteconfig-sh.lua`

The customer dashboard can push config changes to your server. This **server-side** file is the consent
gate for the sensitive ones — it lives on your box, so the dashboard can never flip it for you. By default
the dashboard may change everything **except** three categories.

```lua
RemoteConfig.allowSensitive = false   -- master switch for the sensitive categories below (safe default)

RemoteConfig.sensitive = {
    destinations = false,   -- upload targets / webhook URLs (phone camera, model library, log hooks)
    anticheat    = false,   -- anti-cheat thresholds + actions (off = dashboard can READ but not change them)
    roles        = false,   -- the Discord role IDs in cfg-server-sv.lua that grant staff/dev access
}

-- Extra belt for `destinations`: even when enabled, a pushed URL must be https AND its host must be here.
RemoteConfig.allowedHosts = { "discord.com", "discordapp.com" }   -- empty = any https host (never http)
```

<Callout type="warning">
  Leave `allowSensitive = false` unless you specifically want the dashboard to manage upload destinations,
  anti-cheat, or your staff role IDs — and even then, open only the one category you need. `allowedHosts`
  keeps the blast radius small: a pushed URL must be `https` and, unless the list is empty, on one of these
  hosts.
</Callout>

## Identity link — `configs/cfg-identitylink-sh.lua`

Posts a player's stable identifiers + Discord ID to LACORE's write-only ingest service (once per player
per session), so the [global-ban network](/configuration/gameplay-modules) can match them. No token, keys
or extra `server.cfg` lines — it just works when players are Discord-linked.

```lua
IdentityLink.enabled = true            -- false = nothing is ever sent
IdentityLink.types   = {               -- which identifier types to store (mirrors the ban-match types)
    license = true, license2 = true, discord = true, steam = true, live = true, xbl = true, fivem = true,
}
IdentityLink.requireDiscord = true     -- only store players with a linked Discord (the key the bot looks up by)
```

## Big Brother (admin logging) — `configs/cfg-bigbrother-sv.lua`

Server-only. Every important event flows through `BBLog()` into a searchable DB table and optionally
to Discord. Viewed in-game via `/bblog` (role-gated).

```lua
BigBrother = {
    enabled     = true,
    accessRoles = { "Staff", "Dev", "LeadDeveloper" },   -- who may open /bblog
    retentionDays = 30,     -- logs older than this are pruned on start + daily
    flushSeconds  = 5,      -- buffered writes flush interval …
    maxBuffer     = 250,    -- … or when the buffer exceeds this

    categories = {          -- toggle whole categories on/off
        connection = true,  -- join/leave, identifiers, session length
        combat     = true,  -- kills / deaths
        admin      = true,  -- kick/ban/jail/warn
        anticheat  = true,  -- anticheat flags
        chat       = true,
        command    = true,
        vehicle    = true,  -- spawns / deletes
        mdt        = true,  -- MDT / dispatch actions
    },
}
```

Discord mirroring uses the `lacore_bb_webhook*` convars — see
[Server Convars](/configuration/convars#big-brother-admin-logging).

## Discord Rich Presence — `configs/cfg-presence-sh.lua`

Controls the Discord status players see while playing and the invite link on the connect screen.
Requires a Discord Application (Developer Portal → New Application) for the `appId` and uploaded art
assets. The `Discord`/`Website` buttons appear in the player's Discord profile — great for organic
recruiting. Both the invite and website fall back to the `lacore_discord_invite` / `lacore_website`
convars when left blank.
