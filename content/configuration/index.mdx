# Configuration

LACORE is configured in three layers:

1. **Convars** in `server.cfg` — secrets (Discord token, webhooks) and a few runtime toggles. Secrets
   live here so they never end up in the code or in client downloads. See [Server Convars](/configuration/convars).
2. **`config.lua`** — the main gameplay/world config (agencies, call center, time (`SyncGameTime`), transit, AOPs, …).
3. **`configs/*-sh.lua` / `*-cl.lua` / `*-sv.lua`** — one file per feature. The suffix is the realm:
   `-sh` shared (client **and** server), `-cl` client-only, `-sv` server-only.

<Callout type="info">
  Every file in `configs/` and `config.lua` is **escrow-open** — you can open and edit them freely, and
  most carry a comment header explaining each option. After editing, **restart the resource** for the
  change to apply. Convars keep secrets out of the code and out of client downloads.
</Callout>

## Config file map

Every config file, what it controls, and where it's documented. Files marked *(in-file)* are
self-documented by the comment header at the top of the file.

### Core & world

| File | Scope | Controls |
| --- | --- | --- |
| `config.lua` | shared | [Main config](/configuration/main) — agencies, call center, time (`SyncGameTime`), transit, AOPs |
| `configs/cfg-features-sh.lua` | shared | [Feature Toggles](/configuration/features) — enable/disable any component (e.g. run only the MDT/CAD) |
| `configs/cfg-server-sv.lua` | server | [Discord & Duty](/configuration/discord) — devmode, Discord auth, duty roles, webhooks |
| `configs/cfg-branding-sh.lua` | shared | [Branding & Backup](/configuration/branding) — your community/server name everywhere |

| `configs/cfg-blips-cl.lua` | client | [World, blips & props](/configuration/world) — map blips |
| `configs/cfg-areas-cl.lua` | client | [World, blips & props](/configuration/world) — area labels / zones |
| `configs/cfg-emojis-sh.lua` | shared | [Gameplay modules](/configuration/gameplay-modules) — emoji shortcodes used in chat |

## Localisation

All user-facing strings are localised:

- **Lua** (server/client): `T("key")` → `lang/en.lua` (+ `de.lua`, `ru.lua`).
- **NUI**: `t("key")` → `lang/en.json` (+ `de.json`, `ru.json`).

The active language is set by `Language` in `config.lua`; the MDT/NUI can use a separate language via
`MdtLanguage`. To add a language, copy the `en` files, translate, and register the code in
`configs/locale-sh.lua`.
