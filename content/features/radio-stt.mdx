# Radio Transcript — Speech-to-Text

<Callout type="warning">
  **Experimental — disabled by default.** This feature ships with `STT.enabled = false`. Offline
  recognition accuracy varies by client and it isn't production-ready yet. The full implementation is
  included; enable it in `configs/cfg-stt-sh.lua` to try it.
</Callout>

Units can **speak on the radio and have it transcribed** into a shared, searchable log — powered by
**fully offline speech recognition**. No cloud, no API keys, no per-use cost.

<Callout type="info">
  Recognition runs **locally on each client** via a Vosk model in WebAssembly. LACORE does **not**
  use the browser SpeechRecognition API — it has no working speech backend in FiveM's CEF. The Vosk
  model is **not bundled** (Cfx assets can't contain archives) — you host it and point `STT.model`
  at it; each client downloads it once.
</Callout>

## How it works

<Steps>

### Bind the radio key

The push-to-talk key is unbound by default so it never overrides your own choice. Bind it once under
**FiveM → Settings → Keybinds → "Radio: hold to transcribe"**.

### Hold, speak, release

While on duty, **hold** the key and speak. A live **🎙 …** chip shows the partial transcript. On
**release**, the final text is sent to the server.

### It lands in the radio log

The server tags the line with your **callsign + department**, appends it to a rolling (optionally
persisted) log and broadcasts it to everyone.

</Steps>

## The radio log (`/radiolog`)

Open the transcript panel with **`/radiolog`** — a searchable list of radio traffic with timestamps
and **dept-coloured callsign badges** (LAPD blue, LASD amber, EMS/Fire red). Search by callsign or
text. The log is server-authoritative and shared across the Dispatch console and MDT terminals.

## 911/311 call transcription

When a caller and a dispatcher are connected on a 911/311 call, **both sides are transcribed
automatically** while they talk. Each phrase is appended to the **call transcript** — shown **live**
in the call session and **saved to the call log**. Callers speak naturally, so call transcription
always uses **free recognition** (the radio grammar below is not applied to calls). Toggle it with
`STT.calls`.

## Accuracy — model & radio grammar

Two levers, both built in:

- **The model (self-hosted).** The model is **not shipped with the resource** — Cfx Keymaster
  rejects assets containing archives. Host a Vosk model `.tar.gz` yourself (any static host / CDN)
  and set `STT.model` to its URL. Recommended: `vosk-model-en-us-0.22-lgraph` — bigger and more
  accurate than the tiny "small" models, and it supports the radio grammar below. Each client
  downloads it once and caches it.
- **A radio grammar (the big win for radio).** `STT.grammar` constrains the **push-to-talk radio**
  vocabulary to ten-codes, the phonetic alphabet and common jargon — dramatically improving accuracy
  for codes. Keep `"[unk]"` in the list so words outside it still recognise. This requires a
  `-lgraph` model and is **not** applied to 911/311 calls.

<Callout type="info">
  You can't practically re-train a Vosk model — but the grammar above is the domain-adaptation lever
  that gets you most of the way for radio traffic, without any training.
</Callout>

## Configuration

Everything lives in `configs/cfg-stt-sh.lua`:

```lua
STT.enabled = true      -- master switch
STT.calls   = true      -- auto-transcribe 911/311 dispatcher calls (both sides)
STT.lang    = "auto"    -- "auto" follows each client's UI locale, or "de"/"en"/"ru"
STT.model   = ""        -- REQUIRED to enable: URL to a self-hosted Vosk model .tar.gz (not bundled)
STT.grammar = { "[unk]", "code four", "ten four", "adam", "boy", ... } -- radio (PTT) vocabulary
STT.maxLen  = 220       -- max characters per transmitted line
STT.logSize = 60        -- how many recent lines are kept + sent on open
STT.persist = true      -- keep the log across restarts
STT.store   = "data/radio_log.json"
```

## Notes & limits

- **On-duty only.** Radio transmission is server-gated to units with a callsign; civilians can't
  transmit. Call transcription is gated to the call's participants.
- **Graceful fallback.** A client where the offline engine can't initialise simply can't transmit
  (no crash) — it can still read the log. Verify a client with **`/sttcheck`**.
- **Accuracy** depends on the model, the radio grammar, and the speaker's mic/accent.
- Recognition uses each client's **own CPU** — there is no server-side transcription cost. The
  self-hosted model is downloaded once per client (~128 MB for lgraph) and cached.
