# Web Dispatcher (external portal)

Beyond the in-game console, LACORE can feed an **external browser dispatcher** so a dispatcher can
work from a second screen or off the server entirely.

## How it works

```
FiveM (lacore)  ──▶  Node bridge  ──▶  Web app (browser)
      ▲                                     │
      └──────────  actions relayed  ◀───────┘
```

- The FiveM resource pushes live incident / unit / caller data to a **Node bridge**.
- The bridge broadcasts over WebSocket to the **web app** and serves the app + map tiles from one
  origin.
- Dispatcher actions in the browser are relayed back to FiveM.

## Auth

The portal is gated by **Discord OAuth + role checks**, so only authorised dispatchers can log in.

## Deployment

The bridge ships with a Docker Compose + Caddy (automatic HTTPS) setup for a plug-and-play deploy.
Configure the endpoint with the `lacore_bridge_url` / `lacore_bridge_token` convars.

<Callout type="warning">
  The web dispatcher is optional. The in-game `/dispatch` console works fully on its own — the
  portal is for teams that want an out-of-game dispatcher.
</Callout>
