# CCTV Surveillance

A full surveillance suite for LEO / Security: a fixed-position camera viewer with a **targeting
scanner**, **runtime-placeable field cameras**, and **dispatch-map integration**.

## The viewer (`/cctv`)

Opens a scripted camera at each configured position. The camera does not move — it's a wall camera.

| Control | Action |
| --- | --- |
| **W A S D** / arrow keys | Look around (A/D pan, W/S tilt) |
| **Scroll** | Zoom |
| **Left-click** | Scan the target under the cursor |
| **Backspace / Esc** | Exit |

On open, the operator is streamed to the camera (a placeholder clone stays at the desk) so the area
and its entities load and can be scanned. You cannot fire your weapon while operating a camera.

## The scanner

Left-clicking a **person** or **vehicle** runs a raycast and starts a scan:

<Steps>

### Targeting

A bracket reticle locks onto the target with a sweeping scan line and a live percentage counter.
The scan takes `CCTV.scanTime` (default **6 s**, tune 5000–8000 ms).

### Reveal

An info card fades in and follows the target on screen, showing a compact **mugshot** and — for
registered people — an **MDT summary** (priors count, active BOLOs, recent records). Flags surface
**ACTIVE WARRANT**, **BOLO ACTIVE**, **REPORTED STOLEN** and **UNREGISTERED**.

</Steps>

<Callout type="info">
  NPCs and unregistered targets resolve locally so a scan never hangs; the server only enriches
  **registered** players and plates. Record lookup is gated to LEO / Security.
</Callout>

## Field cameras

Place cameras anywhere at runtime:

- `/camtool` — opens a **placement freecam**: fly around with the mouse + **WASD** (Space / Ctrl up-down,
  Shift = faster). A **translucent preview** of the selected model sticks to the surface you look at —
  ground for poles, walls for cams. **Scroll** changes the model, **←→** rotate it, **↑↓** nudge its
  height, **LMB** places it (exactly where the preview sits), **Backspace** exits. A **crosshair** and a
  ground marker show where the camera will land, and the freecam **collides** with walls so it won't clip
  through them.
- `/cameras` — a manager panel to rename / delete placed cameras, jump the live view to one, and
  toggle the tool.

Placed cameras are persisted server-side (`data/cctv_cams.json`), gated to LEO / Security, and appear
in `/cctv` under the **Field Cameras** networks.

### Physical camera props

Each placed field camera spawns a **real CCTV prop** in the world at that spot (facing where you stood),
visible to **every** player — not just operators. Pick the model while placing (scroll through dome / wall
/ bullet / bank / pole cams). The prop despawns when the camera is removed.

```lua
CCTV.placeProps  = true              -- false = invisible cameras (coordinates only, the old behaviour)
CCTV.defaultProp = "prop_cctv_cam_02a"
CCTV.props = {                        -- the models you can cycle through (only these may be spawned)
  { model = "prop_cctv_cam_02a", label = "Dome cam" },
  { model = "prop_cctv_pole_02", label = "Cam pole 2", camZ = 2.7 },  -- lens height above the base
  -- … add / remove freely
}
```

`camZ` is the height of the lens above the prop's base — wall / dome cams *are* the lens (0), while a
**pole** puts the camera on top, so the live `/cctv` view is rendered from up there. Tune it per model if
a camera's view sits too low or high.

<Callout type="info">
  Only models listed in `CCTV.props` can be spawned — the server rejects anything else, so a client can't
  make everyone load an arbitrary model.
</Callout>

### Camera groups (by locality)

Placed field cameras are **auto-grouped by the in-game locality** they sit in — *Vinewood*,
*Sandy Shores*, *Del Perro*, and so on. Each group becomes its own network in the `/cctv` viewer and its
own section in the `/cameras` manager, so a long camera list stays organised.

- The locality is resolved automatically the moment you place a camera.
- In `/cameras` you can re-assign a camera's group by hand with the 🏷 button.
- **Existing cameras are migrated fully automatically** — after a core update, the first player who
  finishes loading resolves every ungrouped legacy camera into its locality group in one batch (no
  operator has to open CCTV, and no job is required — resolving a locality from coordinates isn't
  privileged). Every open viewer / manager / map updates the moment it's done.

Turn grouping off (one flat "Field Cameras" network) or rename the prefix in
`configs/cfg-cctv-sh.lua`:

```lua
CCTV.autoGroupField  = true     -- false = one flat "Field Cameras" network
CCTV.fieldGroupPrefix = "Field"  -- label prefix, e.g. "Field · Vinewood"
```

## Dispatch map

`/dispatch open` shows every camera you may access as 📹 markers. Click one → **"view this
camera?"** → the console closes, the live CCTV view opens in front, and the console reopens
automatically when you leave the camera.

## Configuration

Cameras and controls live in `configs/cfg-cctv-sh.lua`:

```lua
CCTV.networks = {
  {
    id = 'cityhall', label = 'City Hall', access = 'leo',
    cameras = {
      { id = 'ch_lobby', label = 'Lobby',
        pos = vector3(-545.0, -204.0, 45.0), look = vector3(-545.0, -210.0, 42.0), fov = 55.0 },
    },
  },
}

CCTV.rotSpeed = 70.0   -- look speed (deg/s)
CCTV.scanTime = 6000   -- scan duration (ms)
CCTV.zoomMin  = 20.0
CCTV.zoomMax  = 70.0
```

<Callout type="warning">
  The sample cameras are placeholders — move `pos` / `look` to real locations on your map.
</Callout>
